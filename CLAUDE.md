# Project rules

## GitHub publishing policy (set by repo owner)

- **Private by default.** Do not publish anything publicly on GitHub without the
  owner's explicit, per-case approval. This covers new repositories, branches,
  pull requests, and any commit that contains project content.
- **Deliver finished work as files/bundles** to the user instead of pushing it
  to a public remote.
- **Existing public repos stay public, as-is** — these are the only exceptions:
  - `esejowsky/esejowsky` — profile README (must stay public to render on the GitHub profile)
  - `esejowsky/simba-runner` — public game
- If a task would otherwise require a public push, stop and ask first.
